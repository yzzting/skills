# Change Log

Record high-signal repository changes here. Keep the newest entries first.

## Entry Template

### YYYY-MM-DD - Short Title

- Type: feature | bugfix | refactor | docs | architecture | ops
- Summary: what changed and why it matters
- Affected areas: files, modules, or docs touched
- Verification: tests, manual checks, or review steps performed
- Follow-up: `none` or the next action

## Logging Rules

- add an entry for architecture changes, workflow changes, interface changes, cross-module features, migrations, and durable bug fixes
- skip typo-only edits and trivial cleanup unless they affect operating expectations
- if the change resolved a docs/code mismatch, say so explicitly
- if the change followed a brief or spec, reference it
